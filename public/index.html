Api Docs

GET routes

/status route

Gives back a JSON object with 4 objects:

  • temp → Temperature measured value and currently configured threshold
  • gas → Gas measured value and currently configured threshold
  • alert → Alert status and operational status
  • water → Water Pump status and operational status
Test It → Status

Authentication

To authenticate every post method must have a Auth-Token key-value pair on the request body.

The token must be provided by the administrator.

POST routes

/thr route

Has two children routes: /temp and /gas

Posting a JSON in the body with a "set" object will configure threshold of temperature (/temp) or gas (/gas), to the value of "set", accepts doubles.

/set route

Has two children routes: /alrt and /wtr

Posting a JSON in the body with a "set" object will configure the functioning of the alarm (/alrt) or water pump (/wtr), to the value of "set", must be a boolean.

/store route

Posting a JSON in the body with a "token" object will configure the FCM function (message push service) to your device.

This route should be used only by iOS or Android apps implementing Firebase Cloud Message.
